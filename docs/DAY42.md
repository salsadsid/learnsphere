# Day 42 Learning Notes

## Why this implementation is needed
- Students need a focused browsing experience.
- Category filters reduce choice overload.
- Filter controls improve discovery.

## Advantages
- Faster browsing by topic.
- Clear separation of published vs draft content.
- Lightweight UI enhancements.

## Disadvantages
- Category list depends on existing data.
- No multi-filter support yet.
- Limited to simple query matching.

## Alternatives and tradeoffs
- Use tag-based filtering later.
- Add server-side faceting for scalability.

## Concepts and terms
- Catalog: the student-facing course list.
- Facet: filter option based on data attributes.

## Fundamentals
- Keep filters responsive and clear.
- Default to published courses for students.
