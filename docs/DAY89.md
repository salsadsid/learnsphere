# Day 89 Learning Notes

## Why this implementation is needed
- Final polish reduces last-minute surprises before deployment.

## Advantages
- Visible updates improve user trust.
- Clear staging steps reduce release risk.

## Disadvantages
- Polish can expand scope late in the cycle.
- Staging environments add maintenance overhead.

## Alternatives and tradeoffs
- Ship minimal polish and iterate post-launch (faster, riskier).
- Use feature flags for staged rollout (more control, more effort).

## Concepts and terms
- Staging deploy: pre-production release for validation.
- Release readiness: checklist-driven validation before launch.

## Fundamentals
- Treat staging like production to catch issues early.
